# ADR-0001: Record Architecture Decisions

## Status
Accepted

## Context
Necesitamos registrar las decisiones arquitectónicas importantes del proyecto para mantener un historial y contexto de por qué se tomaron ciertas decisiones.

## Decision
Usaremos Architecture Decision Records (ADR), como se describe por Michael Nygard en este artículo: http://thinkrelevance.com/blog/2011/11/15/documenting-architecture-decisions

## Consequences
- Los ADRs se guardarán en `docs/architecture/decisions/`
- Cada ADR tendrá un número secuencial y un título descriptivo
- Los ADRs se escribirán en Markdown
- Cualquier decisión arquitectónica significativa debe tener un ADR
